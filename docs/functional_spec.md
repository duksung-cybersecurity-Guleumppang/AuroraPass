# 수강신청 시스템 기능 명세

이 문서는 이미지 예시를 기반으로 수강신청 시스템의 전체 기능 흐름과 각 단계별 요구사항을 정의합니다.

## 전체 시스템 흐름

사용자는 다음 순서에 따라 시스템을 이용합니다.

1.  **로그인 페이지 접속 및 CAPTCHA 인증 (`1.png` -> `2.png` -> `3.png`)**
    -   사용자가 로그인 페이지에 접속하면 CAPTCHA 인증이 먼저 요구됩니다.
    -   CAPTCHA 인증을 성공적으로 완료해야 로그인 버튼이 활성화됩니다.
2.  **로그인 및 수강신청 페이지 이동 (`3.png` -> `4.png`)**
    -   활성화된 버튼을 통해 로그인하면, 수강신청 페이지로 이동합니다.
3.  **수강신청 및 비정상 접근 탐지 (`4.png` -> `5.png`)**
    -   사용자는 강의를 선택하고 수강신청을 시도합니다.
    -   만약 비정상적인 요청(예: 짧은 시간 내 5회 이상 클릭)이 감지되면, 시스템은 추가 CAPTCHA 인증을 요구합니다.

---

## 1. 사용자 인증 (로그인)

### 화면 구성

-   **초기 상태 (`1.png`, `2.png`)**:
    -   아이디, 비밀번호 입력 필드가 표시됩니다.
    -   오디오 또는 이미지 형태의 CAPTCHA 문제가 제시됩니다.
    -   '로그인' 버튼은 비활성화 상태입니다.
-   **CAPTCHA 인증 성공 후 (`3.png`)**:
    -   '로그인' 버튼이 활성화됩니다.

### 기능 요구사항

1.  **CAPTCHA 생성 (Front-end & Back-end)**
    -   **요청**: 페이지 로드 시, 클라이언트는 서버에 CAPTCHA 생성을 요청합니다. (`GET /api/captcha/generate`)
    -   **응답**: 서버는 CAPTCHA 문제(오디오 경로 등)와 고유 ID(`captchaId`)를 반환합니다.
2.  **CAPTCHA 검증 (Front-end & Back-end)**
    -   **사용자 입력**: 사용자가 CAPTCHA 정답을 입력합니다.
    -   **요청**: 클라이언트는 `captchaId`와 사용자 입력값을 서버로 전송하여 검증을 요청합니다. (`POST /api/captcha/verify`)
    -   **처리**:
        -   **성공 시**: 서버는 성공을 알리고, 클라이언트는 '로그인' 버튼을 활성화합니다.
        -   **실패 시**: 서버는 실패를 알리고, 클라이언트는 에러 메시지를 표시하며 새 CAPTCHA를 요청합니다.
3.  **로그인 처리 (Front-end & Back-end)**
    -   **요청**: 사용자가 활성화된 '로그인' 버튼을 클릭하면, 클라이언트는 아이디와 비밀번호를 서버로 전송합니다. (`POST /api/users/login`)
    -   **처리**: 서버는 사용자 정보를 확인하고, 성공 시 세션 또는 토큰을 발급하여 로그인 상태를 유지합니다.

## 2. 수강신청

### 화면 구성 (`4.png`)

-   전체 개설 강의 목록이 표시됩니다. (과목명, 교수, 시간, 정원 등)
-   사용자가 선택한 강의를 담을 수 있는 '장바구니' 영역이 있습니다.
-   최종적으로 수강신청을 확정하는 '수강신청' 버튼이 있습니다.

### 기능 요구사항

1.  **강의 목록 조회**: 페이지 진입 시, 개설된 전체 강의 목록을 서버에서 불러와 표시합니다. (`GET /api/courses`)
2.  **장바구니 관리**:
    -   **추가**: 특정 강의를 장바구니에 추가합니다. (`POST /api/cart`)
    -   **삭제**: 장바구니에서 특정 강의를 제거합니다. (`DELETE /api/cart/{courseId}`)
    -   **조회**: 현재 장바구니에 담긴 목록을 확인합니다. (`GET /api/cart`)
3.  **수강신청 실행**:
    -   **요청**: '수강신청' 버튼을 클릭하면, 장바구니에 담긴 강의 목록을 서버에 전송하여 최종 신청을 요청합니다. (`POST /api/enroll`)
    -   **처리**: 서버는 정원, 시간표 중복 등을 검사하여 수강신청을 처리합니다.

## 3. 비정상 접근 제어

### 화면 구성 (`5.png`)

-   기존 수강신청 화면 위에 CAPTCHA 인증을 요구하는 모달(팝업) 창이 표시됩니다.
-   사용자는 이 CAPTCHA를 통과해야 다음 작업을 진행할 수 있습니다.

### 기능 요구사항

1.  **비정상 접근 탐지 (Back-end)**
    -   **조건**: 특정 계정에서 짧은 시간 내에 '수강신청' 요청(`POST /api/enroll`)이 5회 이상 발생하는 등 비정상적인 패턴이 감지됩니다.
    -   **처리**: 서버는 수강신청을 처리하는 대신, CAPTCHA 인증이 필요하다는 특별한 에러 코드를 클라이언트에 반환합니다.
2.  **CAPTCHA 화면 표시 (Front-end)**
    -   클라이언트는 서버로부터 CAPTCHA 요구 응답을 받으면, `5.png`와 같이 인증 모달 창을 화면에 표시합니다.
3.  **인증 후 작업 재개**
    -   사용자가 모달 창의 CAPTCHA를 성공적으로 통과하면, 이전에 시도했던 수강신청을 다시 실행할 수 있는 권한을 일시적으로 부여받습니다.
